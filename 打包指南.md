# Android APK 打包完整指南

## ⚠️ 重要说明
**buildozer 只能在 Linux 系统上运行，Windows 无法直接使用！**

---

## 方式一：GitHub Actions 自动构建（最推荐）

### 步骤：

#### 1. 创建 GitHub 账号
访问 https://github.com 注册账号（如果已有可跳过）

#### 2. 创建新仓库
- 点击右上角 "+" → "New repository"
- 仓库名称：`class-reminder`
- 选择 "Public" 或 "Private"
- 点击 "Create repository"

#### 3. 上传代码
**方法A - 网页上传（简单）：**
- 在仓库页面点击 "uploading an existing file"
- 将 `apk` 文件夹内的所有文件拖拽上传
- 点击 "Commit changes"

**方法B - Git 命令行：**
```powershell
cd C:\Users\Administrator\Desktop\class_a\apk
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/你的用户名/class-reminder.git
git push -u origin main
```

#### 4. 下载 APK
- 点击仓库顶部的 "Actions" 标签
- 等待构建完成（约20-30分钟）
- 点击构建记录，在页面底部 "Artifacts" 下载 `class-reminder-apk.zip`
- 解压得到 `.apk` 文件

## 方式二：安装 WSL2 本地打包

### 步骤：

#### 1. 安装 WSL2
以**管理员身份**打开 PowerShell，运行：
```powershell
wsl --install -d Ubuntu-22.04
```
安装完成后重启电脑。

#### 2. 进入 WSL
```powershell
wsl
```

#### 3. 安装依赖
```bash
sudo apt update
sudo apt install -y git zip unzip openjdk-17-jdk python3-pip autoconf libtool pkg-config libncurses5-dev libncursesw5-dev libtinfo5 cmake libffi-dev libssl-dev automake

pip3 install buildozer cython==0.29.33
```

#### 4. 复制项目到 WSL
```bash
cp -r /mnt/c/Users/Administrator/Desktop/class_a/apk ~/class_reminder
cd ~/class_reminder
```

#### 5. 打包 APK
```bash
buildozer android debug
```
首次打包需要下载 SDK/NDK，耗时约30-60分钟。

#### 6. 获取 APK
APK 位于 `~/class_reminder/bin/` 目录，复制到 Windows：
```bash
cp ~/class_reminder/bin/*.apk /mnt/c/Users/Administrator/Desktop/class_a/
```

## 方式三：使用在线 Linux 服务器

如果你有云服务器（阿里云、腾讯云等），可以：

```bash
# 安装依赖
sudo apt update
sudo apt install -y git zip unzip openjdk-17-jdk python3-pip autoconf libtool pkg-config libncurses5-dev libncursesw5-dev libtinfo5 cmake libffi-dev libssl-dev automake

# 上传代码并打包
git clone 你的仓库地址
cd class-reminder/apk
buildozer android debug
```

## 安装 APK 到手机

1. 将 APK 传输到手机（数据线/微信/QQ/云盘）
2. 在手机上打开 APK 文件
3. 如提示"禁止安装"，前往设置开启"允许安装未知来源应用"
4. 安装完成后打开应用

## 常见问题

### Q: GitHub Actions 构建失败怎么办？
A: 检查 Actions 页面的错误日志，通常是依赖版本问题。

### Q: WSL 安装失败？
A: 确保已启用虚拟化，BIOS 中开启 VT-x/AMD-V。

### Q: buildozer 下载 SDK 很慢？
A: 可以配置国内镜像，或使用 GitHub Actions。
